# ⚠️ Erros Conhecidos e Como Resolver

## 1. ❌ CRÍTICO: Missing or insufficient permissions

### Erro Completo
```
Error fetching posts: FirebaseError: [code=permission-denied]: Missing or insufficient permissions.
Error creating post: FirebaseError: [code=permission-denied]: Missing or insufficient permissions.
```

### ✅ SOLUÇÃO (OBRIGATÓRIA)

**Este erro IMPEDE o funcionamento do sistema.**

Você **DEVE** configurar as regras do Firestore no Firebase Console:

1. Acesse: https://console.firebase.google.com/
2. Selecione o projeto **vera-mei**
3. Vá em **Firestore Database** > **Rules**
4. Cole estas regras de desenvolvimento:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

5. Clique em **"Publicar"**
6. Aguarde 30 segundos
7. Recarregue o aplicativo

**📖 Leia o arquivo FIRESTORE-RULES.md para instruções detalhadas.**

## 2. ⚠️ INFORMATIVO: bcrypt module not found

### Erro Completo
```
event loop error: Error: Cannot find module '/var/tmp/sb-compile-edge-runtime/node_modules/localhost/bcrypt/5.1.1/lib/binding/napi-v3/bcrypt_lib.node'
Require stack:
- /var/tmp/sb-compile-edge-runtime/node_modules/localhost/bcrypt/5.1.1/bcrypt.js
```

### ✅ SOLUÇÃO

**Este erro DEVE ser IGNORADO. Ele não pode ser corrigido e não afeta o sistema.**

**Por que acontece:**
- O Figma Make compila automaticamente arquivos Supabase Edge Functions
- O arquivo `/supabase/functions/server/kv_store.tsx` é **AUTOGENERATED** e **PROTEGIDO**
- Este arquivo importa `@supabase/supabase-js` que depende de `bcrypt`
- O `bcrypt` requer módulos nativos (`.node`) que não existem no ambiente Edge Runtime
- O sistema **NÃO USA** Supabase - foi migrado para Firebase
- **Não é possível deletar ou modificar** arquivos protegidos do Figma Make

**O que fazer:**
- ✅ **IGNORE completamente este erro**
- ✅ O sistema funciona perfeitamente com Firebase
- ✅ Este erro NÃO afeta nenhuma funcionalidade
- ✅ Aparece apenas no console do servidor (event loop), não no frontend
- ✅ É um resíduo da infraestrutura Supabase do Figma Make
- ✅ Não há ação necessária

**Confirmação de que é seguro ignorar:**
- ✅ Autenticação funciona (Firebase Auth)
- ✅ Posts são criados/editados/deletados (Firestore)
- ✅ Imagens são enviadas
- ✅ Blog público funciona
- ✅ Painel admin funciona

**Status:** Erro cosmético inevitável, **ZERO impacto no funcionamento**.

## 3. ⚠️ INFORMATIVO: findDOMNode is deprecated

### Erro Completo
```
Warning: findDOMNode is deprecated and will be removed in the next major release. Instead, add a ref directly to the element you want to reference.
```

### ✅ SOLUÇÃO

**Este warning pode ser IGNORADO com segurança.**

**Por que acontece:**
- A biblioteca **react-quill** (editor de texto) usa `findDOMNode`
- É uma API antiga do React que será removida no futuro
- É um problema da **biblioteca**, não do nosso código

**O que fazer:**
- ✅ **IGNORE este warning**
- ✅ Não afeta o funcionamento do editor
- ✅ Será corrigido quando react-quill atualizar

**Status:** Warning da biblioteca externa, sem impacto.

## 4. 🔧 Firebase: User not logged in after signup

### Erro
Após criar conta, não consigo criar posts.

### ✅ SOLUÇÃO

1. **Faça logout** e **login novamente**
2. Certifique-se de que as regras do Firestore estão configuradas
3. Verifique no console do navegador se o usuário está autenticado:

```javascript
// No console do navegador (F12)
firebase.auth().currentUser
// Deve retornar um objeto com uid, email, etc.
```

## 5. 🔧 Posts não aparecem após criar

### Erro
Criei um post mas ele não aparece na lista.

### ✅ SOLUÇÕES

**Solução 1: Aguarde alguns segundos**
- Firestore pode levar 1-3 segundos para sincronizar
- Recarregue a página após alguns segundos

**Solução 2: Verifique as regras do Firestore**
- Se o post foi criado mas não aparece, pode ser problema de permissão de leitura
- Veja o arquivo FIRESTORE-RULES.md

**Solução 3: Limpe o cache**
- Pressione `Ctrl+Shift+R` (Windows/Linux) ou `Cmd+Shift+R` (Mac)
- Isso força o recarregamento completo

## 6. 🔧 Erro ao fazer upload de imagem

### Erro
```
Error uploading image
```

### ✅ SOLUÇÃO

**Verifique o tamanho da imagem:**
- Máximo: **5MB por imagem**
- Formatos aceitos: JPG, PNG, GIF, WebP

**Se a imagem é muito grande:**
1. Use um compressor online (ex: TinyPNG)
2. Reduza a resolução da imagem
3. Converta para formato mais eficiente (WebP)

**Nota:** Imagens são armazenadas em base64 no Firestore, o que aumenta o tamanho em ~33%.
