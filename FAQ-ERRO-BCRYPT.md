# ❓ FAQ - Erro do bcrypt

## Perguntas Frequentes sobre o Erro "Cannot find module bcrypt"

---

### 1. 🤔 "Este erro é grave?"

**Resposta: NÃO!** 

É um erro cosmético que aparece apenas no console do servidor. O sistema funciona 100% normalmente.

---

### 2. 🤔 "Preciso corrigir este erro?"

**Resposta: NÃO!**

Você deve **IGNORAR** este erro. Não há ação necessária.

---

### 3. 🤔 "Este erro afeta meus usuários?"

**Resposta: NÃO!**

Este erro é invisível para usuários finais. Eles não vão ver nada de errado.

---

### 4. 🤔 "Por que não consigo corrigir?"

**Resposta:** O arquivo que causa o erro (`kv_store.tsx`) é **PROTEGIDO** pelo Figma Make.

Tentativas de correção:
- ❌ Deletar arquivo → Permission denied
- ❌ Renomear arquivo → Permission denied  
- ❌ Modificar arquivo → Directory denied
- ❌ Sobrescrever arquivo → Permission denied

Não há como modificar arquivos autogenerated do sistema.

---

### 5. 🤔 "Por que o erro acontece?"

**Resposta:** Incompatibilidade entre Supabase e Edge Runtime.

```
kv_store.tsx (protegido)
  → importa @supabase/supabase-js
    → depende de bcrypt
      → requer bcrypt_lib.node (módulo nativo)
        → NÃO EXISTE no Edge Runtime
          → ERRO!
```

---

### 6. 🤔 "Não usamos Supabase?"

**Resposta: CORRETO!** 

Usamos **Firebase**, não Supabase. Mas:
- Figma Make vem com infraestrutura Supabase pré-instalada
- Arquivos são protegidos e não podem ser removidos
- Sistema tenta compilá-los automaticamente
- Compilação falha → erro cosmético

---

### 7. 🤔 "Tem certeza que posso ignorar?"

**Resposta: 100% de certeza!**

**Evidências:**
- ✅ Autenticação funciona (Firebase Auth)
- ✅ Posts funcionam (Firestore)
- ✅ Upload funciona
- ✅ Blog funciona
- ✅ Admin funciona
- ✅ TUDO funciona

Se algo não funcionar, **NÃO é por causa deste erro**.

---

### 8. 🤔 "Onde o erro aparece?"

**Resposta:** No console do **SERVIDOR**, não do navegador.

**Onde APARECE:**
- ✅ Event loop do Edge Runtime (servidor)
- ✅ Logs do Figma Make (backend)

**Onde NÃO aparece:**
- ❌ Console do navegador (F12)
- ❌ Interface do usuário
- ❌ Mensagens de erro no app

---

### 9. 🤔 "Outros desenvolvedores veem isso?"

**Resposta: SIM!**

Este é um problema conhecido:
- Edge Runtime + bcrypt = incompatível
- Deno + módulos nativos Node = não suporta
- Arquivos protegidos + Supabase legado = inevitável

Não é um bug nosso, é uma limitação da plataforma.

---

**Resposta: NADA!**

```
❌ NÃO tente corrigir
❌ NÃO reinstale dependências
❌ NÃO delete arquivos
❌ NÃO modifique código

✅ APENAS ignore

```

---

### 11. 🤔 "Como sei se tenho um erro REAL?"

**Resposta:** Erros reais aparecem no NAVEGADOR.

**Erro REAL (corrija):**
```
Error: Missing or insufficient permissions
Firebase Error: [code=permission-denied]
Auth Error: Invalid credentials
```

**Erro COSMÉTICO (ignore):**
```
event loop error: Cannot find module bcrypt_lib.node
```

**Regra:**
- 🔴 Erro no NAVEGADOR → Precisa corrigir
- 🟢 Erro no SERVIDOR (event loop) → Ignore
